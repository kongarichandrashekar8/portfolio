Chat Bot

Main aim of this project is to provide 24/7 fast services to students

For Example

  • If student what to know when is the last day to pay sem fee.
  • which documents requried for fee reimbursement
  • Mid exam dates
  • particular book available in library or not etc

By using this chat bot students can get clear and fast information about the query,No space for confusion

Steps:

  1. Student login with his credentials
  2. In backend we check he/she is valid user or not
  3. if he/she is valid user then ,chat bot provides services
  4. Student can enter his/her query
  5. If that query is matches with any of the intent, chat bot will provide solution directly
  6. If the query not matching any of the intent then chat bot provide some categories ,student has to select the issue in that list of categories
  7. Trouble ticket will be generated and stored in database after selection of issue and that Trouble ticket will show to student
  8. With that Trouble ticket number he/she can checks the status of the issue when ever he/she wants
  9. by typing logout student can logout from chatbot

DialogFlow

  1. Dialogflow is a natural language understanding platform used to design and integrate a conversational user interface into mobile apps, web applications, devices, bots, interactive voice response systems and related uses.
  2. An “agent” in Dialogflow is the the same as one chatbot.
  3. An intent is an action a user wants to do or a question a user has. Example : Order a pizza
  4. Entities are facts in a sentence, that refer to the who, what, when and where.Example : I want to order a veggie pizza.
  5. a webhook can be used to fetch data from your server whenever a certain intent having webhook enabled is invoked.

MongoDB

For storing customer data I used MongoDb ,because it is very efficient when there are fewer realations to manage,It is scalable.In this project user data has no inner relations

  1. Instead of using tables and rows as in the traditional relational databases, MongoDB makes use of collections and documents. Documents consist of key-value pairs which are the basic unit of data in MongoDB.
  2. Data is stored in MongoDB in the form of JSON Style.
  3. To interact with MongoDB Mongoose is used
  4. It is an Object Data Modeling(ODM) library for MongoDB
  5. Using mongoose ,a user can define the schema for the document in a particular collection and we can manipulate
